System Logging and Forensics

I have learned from this section that logging is the value of troubleshooting, diagnosing, and isolating the cause of the security incident. Without loggings, the security engineer will struggle to find what happens during the incident or the attack. Different types of logging are critical to the organisations starting with the system logs to the application logs and ending with the network, firewall, and security monitoring tools logs. For example, by checking the firewall logs, the security engineer can know if any malicious staff or attackers hit the firewall policies. In addition, alerting systems in security applications like the IPS and IPS can help investigate any security incidents.
